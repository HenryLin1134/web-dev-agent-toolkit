# Web-Creator 行為對比圖

## 📊 改進前後對比

### 場景：使用者要求建立登入頁面

```
使用者輸入：
"@web-creator 我想要一個登入頁面"
```

---

## ❌ 改進前（v1.0.0）- 一般 Agent 模式

```
┌─────────────────────────────────────────────┐
│  使用者: "我想要一個登入頁面"                │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  ❌ Agent 直接回應                          │
│                                             │
│  "好的，我來幫你建立登入頁面。              │
│   你需要以下檔案：                          │
│   - login.html                              │
│   - login.css                               │
│   - login.js                                │
│                                             │
│   這是 HTML 程式碼：                        │
│   <form>...</form>"                         │
└─────────────────────────────────────────────┘
                    ↓
           ❌ 問題：
    ┌──────────────────────┐
    │ • 跳過規劃階段       │
    │ • 沒有建立 CHANGELOG │
    │ • 沒有調用 Agent     │
    │ • 缺乏追蹤性         │
    │ • 品質無法保證       │
    └──────────────────────┘
```

## 🎯 關鍵改進點

### 1. 強制執行機制

```
改進前: 提示建議（容易忽略）
改進後: 強制規則（必須遵守）
```

### 2. 檢查點系統

```
改進前: 沒有檢查點
改進後: 每個步驟都有檢查點
        ✓ CHANGELOG 建立檢查
        ✓ Agent 調用檢查
        ✓ 輸出文檔檢查
```

### 3. 自我修正機制

```
改進前: 無自我檢查
改進後: 內建自我檢查
        IF 行為不正確:
          THEN 停止並重啟正確流程
```

## 💡 使用建議

### 觸發正確行為

**推薦做法**:
```
@web-creator [清楚描述需求]
```

期待看到：
1. 🧠 啟動工作流程宣告
2. 📝 建立 CHANGELOG
3. 📋 調用 Planning Agent
4. ⏭️ 依序執行各階段

### 糾正錯誤行為

如果看到直接回應：
```
"請按照正確的工作流程執行：
1. 建立 CHANGELOG
2. 調用 Planning Agent
3. 依序執行各階段"
```
